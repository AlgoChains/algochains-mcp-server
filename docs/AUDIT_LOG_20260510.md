## P0 Bugs — All Fixed ✅

### BUG-01 — `close_position` confirmation gate missing
- **File:** `algochains-mcp-server/src/algochains_mcp/server.py:5163`
- **Evidence:** Handler fell through to `conn.close_position()` with only a debug log when elicitation was unavailable; no `ALGOCHAINS_REQUIRE_CONFIRMATION` check.
- **Fix:** Elicitation + env gate added. Mirrors `place_order` pattern. When `ALGOCHAINS_REQUIRE_CONFIRMATION=1` and client lacks elicitation, returns `status: blocked`.
- **Status:** FIXED ✅

### BUG-02 — `cancel_order` confirmation gate missing
- **File:** `algochains-mcp-server/src/algochains_mcp/server.py:5104`
- **Evidence:** Zero confirmation machinery. Cancelling a working bracket leg mid-trade causes orphan positions.
- **Fix:** Full elicitation + `ALGOCHAINS_REQUIRE_CONFIRMATION` gate added at `server.py:5104–5145`.
- **Status:** FIXED ✅

### BUG-03 — Phase 3 VIX/daily-loss/circuit-breaker gates fail-open
- **File:** `algochains-control-tower/FUTURES_SCALPER_UPGRADED.py:9353–9395`
- **Evidence:** `except Exception: pass` blocks silently skipped Phase 3 safety gates on any exception.
- **Fix:** All three gates now fail-CLOSED. Exception → log error + `return None`. Confirmed in live file at line 9358.
- **Owner approval:** Explicitly approved.
- **Status:** FIXED ✅

### BUG-04 — `exit_position` failure swallowed silently
- **File:** `algochains-control-tower/FUTURES_SCALPER_UPGRADED.py:16901–16917`
- **Evidence:** `except Exception: pass` after emergency exit cleared PENDING state even when exit failed, leaving open position without bracket.
- **Fix:** `_exit_ok = False` path now sends critical Slack alert. PENDING state not cleared on failure.
- **Owner approval:** Explicitly approved.
- **Status:** FIXED ✅

### BUG-05 — `consecutive_losses` hardcoded to 0
- **File:** `algochains-mcp-server/src/algochains_mcp/server.py:4930–4985`
- **Evidence:** `_consecutive_losses = 0` permanently disabled loss-streak halt gate in `check_all()`.
- **Fix:** Derives from fills API (newest-first scan, trailing loss streak). Reconciliation fallback: `state/signal_health.json`. Both-fail logs WARNING and assumes 0.
- **Note:** Python 3 except-var scope bug fixed — `_fills_err` captured as `_fills_api_err_str` before except block exits.
- **Status:** FIXED ✅

### BUG-06 — Guardrail gate exception handler fails-open
- **File:** `algochains-mcp-server/src/algochains_mcp/security/guardrail.py:203`
- **Evidence:** Any exception in a gate returned `ok=True` ("Gate error skipped"), allowing the order through.
- **Fix:** Exceptions now return `ok=False` and log ERROR. Order is blocked. Gate results include `"Gate BLOCKED (internal error — fail-safe)"` reason.
- **Status:** FIXED ✅

### BUG-07 — Kalshi execution tools mis-tiered as `WRITE_LOCAL`
- **File:** `algochains-mcp-server/src/algochains_mcp/tool_danger_tiers.py`
- **Evidence:** `run_safe_compounder`, `run_kalshi_full_pipeline`, `run_kalshi_strategy_order` inherited `WRITE_LOCAL` via `run_` prefix rule. Autonomous agents (capped at WRITE_LOCAL) could trigger live Kalshi orders without `owner_token`.
- **Fix:** All three explicitly set to `TIER_ORDER_EXEC`. Contract tests extended to cover all three (`tests/test_dynamic_dispatch_safety.py`).
- **Status:** FIXED ✅

## P1 Bugs — All Fixed ✅

### BUG-08 — `check_token` TypeError path returns `False` after successful `authenticate()`
- **File:** `algochains-control-tower/tradovate_client.py`
- **Evidence:** TypeError path (timezone mismatch) called `authenticate()` then returned `False`. Callers treat `False` as "token invalid — block order", causing false order blockage even after successful re-auth.
- **Fix:** Returns `True` after successful `authenticate()`. If `authenticate()` raises, exception propagates (fail-closed).
- **Status:** FIXED ✅

### BUG-09 — 1-lot bypass skips ALL risk checks
- **File:** `algochains-control-tower/trading_safeguards.py:272–287`
- **Evidence:** `qty_int == 1` returned `True` unconditionally, bypassing daily-loss and VIX gates.
- **Fix:** 1-lot fast-path preserved for margin/balance/exposure checks. `pre_trade_safety_check()` (portfolio daily-loss + VIX halt) now runs for all 1-lot orders.
- **Status:** FIXED ✅

### BUG-10 — `launchctl` allowlist too broad
- **File:** `algochains-control-tower/autonomous/adaptive_brain.py:101`
- **Evidence:** Prefix `"launchctl "` allowed `bootout`, `remove`, and other destructive subcommands.
- **Fix:** Narrowed to `"launchctl kickstart "` only. `kickstart -k` is the only safe restart subcommand.
- **Status:** FIXED ✅

### BUG-11 — JWT `verify_signature=False` in adaptive_brain
- **File:** `algochains-control-tower/autonomous/adaptive_brain.py`
- **Evidence:** `jwt.decode(..., verify_signature=False)` trusted token's `exp` claim without signature verification.
- **Fix:** Replaced with raw base64 payload parse (middle segment, no jwt library). Reads `exp` directly from token bytes without trusting the signature.
- **Status:** FIXED ✅

### BUG-12 — `shell=True` + prefix bypass in `generic_skill_executor`
- **File:** `algochains-control-tower/generic_skill_executor.py`
- **Evidence:** `subprocess.run(cmd, shell=True)` with prefix-only allowlist. Semicolons and pipes allowed injection: `grep logs; rm -rf state/`.
- **Fix 1:** Metacharacter block: `;`, `|`, `&`, backtick, `$()`, `<()`, `>()`, newline. (Audit-1 refined from original — standalone `()` no longer blocked.)
- **Fix 2:** `subprocess.run(shlex.split(cmd), shell=False)`. Defense-in-depth.
- **Status:** FIXED ✅

### BUG-13 — VIX fallback chain: 0 treated as valid, logged at debug
- **Files:** `algochains-mcp-server/src/algochains_mcp/server.py`, `algochains-mcp-server/src/algochains_mcp/security/guardrail.py`
- **Evidence:** VIX=0 silently skipped the VIX gate; fetch failures logged at DEBUG (invisible in prod).
- **Fix:** `guardrail.py` `_gate_vix`: VIX=0 logs WARNING and returns `(True, "VIX unknown — gate skipped [WARNING: protection weakened]")`. `server.py`: fetch failures now `logger.warning` with explicit "VIX gate will be SKIPPED" message.
- **Status:** FIXED ✅

### BUG-14 — `vix_regime.json` only enforced when file exists
- **File:** `algochains-control-tower/trading_safeguards.py`
- **Evidence:** Missing `vix_regime.json` silently skipped the VIX halt gate. Absent file = regime fetcher never ran or crashed.
- **Fix:** `else` branch added: if file missing, logs WARNING that VIX halt gate is inactive.
- **Status:** FIXED ✅

### BUG-15 — `submit_institutional_order` bypasses guardrails
- **File:** `algochains-mcp-server/src/algochains_mcp/server.py`
- **Evidence:** `submit_institutional_order` skipped `TradingGuardrails` entirely. `compliance_override=True` bypassed VIX, daily-loss, drawdown, velocity gates.
- **Fix:** Same guardrail pre-flight as `place_order` wired for non-override calls. Broker extracted safely against `_KNOWN_BROKERS` set.
- **Status:** FIXED ✅

## P2 Bugs — All Fixed ✅

### BUG-16 (P2-A) — Decay metrics fetch failure returns synthetic healthy profile
- **File:** `algochains-mcp-server/src/algochains_mcp/builder_sdk/paper_trading_graduation.py`
- **Evidence:** Exception in decay fetch returned `sharpe_30d=1.0, drawdown=0, signals=10` — a strategy with real decay passed promotion checks whenever the decay API was unavailable.
- **Fix:** Fail-closed profile: `sharpe=-1.0`, `max_drawdown_30d_pct=50.0`, `consecutive_losses=999`, `signals_14d=0`.
- **Status:** FIXED ✅

### BUG-17 (P2-B) — `total_trades_copied` increments when order never placed
- **File:** `algochains-mcp-server/src/algochains_mcp/social_trading/copy_engine.py`
- **Evidence:** Counter incremented regardless of whether `_place_alpaca_order` was called. Unsupported brokers silently counted as "copied".
- **Fix:** Counter increments only after successful Alpaca placement. Unsupported brokers log WARNING. No count increment.
- **Status:** FIXED ✅

### BUG-18 (P2-C) — Null `subscriber_id` becomes literal `"None"` string
- **File:** `algochains-mcp-server/src/algochains_mcp/subscriber_auth.py`
- **Evidence:** `str(None)` produced `"None"` as subscriber_id, which could match across unrelated authZ checks.
- **Fix:** Guard added: null/falsy `subscriber_id` returns `None` (unauthenticated) instead of `"None"` string.
- **Status:** FIXED ✅

### BUG-19 (P2-D) — In-memory replay nonces cleared on restart
- **File:** `algochains-mcp-server/src/algochains_mcp/security/replay_guard.py`
- **Evidence:** In-memory `_NONCE_STORE` reset on every process restart, allowing replay attacks within `MAX_AGE_SECONDS` window post-restart.
- **Fix:** Redis-backed SETNX nonce store (configured via `REPLAY_GUARD_REDIS_URL`). Falls back to in-memory with startup WARNING. Module-level mutable `_NONCE_STORE` alias removed.
- **Status:** FIXED ✅

### BUG-20 (P2-E) — Hardcoded HTTP fallback URL in trade propagation
- **File:** `algochains-mcp-server/src/algochains_mcp/trade_propagation.py`
- **Evidence:** `_resolve_url()` fell back to hardcoded `http://172.232.170.168/signals/signal/` (plaintext HTTP) when env unset. HMAC secret and signals transmitted in cleartext.
- **Fix:** Fails closed with `RuntimeError` when `ALGOCHAINS_SIGNAL_URL` unset. Logs WARNING if HTTP URL is configured.
- **Status:** FIXED ✅

### BUG-21 (P2-F) — `DEFI_ENABLED=true` returns `status=ok` with random `tx_hash`
- **File:** `algochains-mcp-server/src/algochains_mcp/defi_engine/dex_aggregator.py`
- **Evidence:** `_DEFI_LIVE=True` caused `execute_swap` to return `status="ok"` with `uuid4()` tx_hash — implying real on-chain execution when none occurred.
- **Fix:** `_DEFI_LIVE` forced to `False`. `ALGOCHAINS_DEFI_ENABLED=true` logs one-time WARNING.
- **Status:** FIXED ✅

### BUG-22 (P2-G) — Circuit-breaker state load failure logged at WARNING, state resets to CLOSED
- **File:** `algochains-mcp-server/src/algochains_mcp/trading_guardrails.py:309–325`
- **Evidence:** Corrupt state file reset all CBs to CLOSED (trading allowed). Log level WARNING was below alerting threshold.
- **Fix (two-part):**
  1. Log level upgraded to ERROR.
  2. All 4 brokers (tradovate, alpaca, oanda, rithmic) now tripped to OPEN with `GuardrailReason.STATE_FILE_CORRUPT`. Operator must call `reset_circuit_breaker()` after verifying state.
- **Status:** FIXED ✅

### BUG-23 (P2-H) — Stale disk token after in-memory renewal
- **File:** `algochains-control-tower/tradovate_client.py`
- **Evidence:** After writing new token to `.env`, `os.environ` not updated. WebSocket reconnect flows reading `os.getenv("TRADOVATE_ACCESS_TOKEN")` got stale token.
- **Fix:** `os.environ["TRADOVATE_ACCESS_TOKEN"] = new_token` added before `.env` write, within `fcntl.flock` context.
- **Status:** FIXED ✅

### BUG-24 (P2-I) — Watchdog `ps` substring match causes false positives
- **File:** `algochains-control-tower/autonomous/autonomous_watchdog.py`
- **Evidence:** `match_str in line` matched unrelated processes (backup viewers, grep processes watching the bot file).
- **Fix:** Requires match_str as standalone token (preceded by space or path separator) AND verifies line is a Python invocation. Skips lines containing "grep".
- **Status:** FIXED ✅

## Audit-of-Implementations Findings — All Fixed ✅

### AUDIT-1 — Metacharacter regex too aggressive (blocked standalone `()`)
- **File:** `generic_skill_executor.py`
- **Issue:** Original regex `[;&|`$()]` blocked `(` and `)` — would reject `python3 -c "print('x')"`.
- **Fix:** Narrowed to injection-only patterns: `;`, `|`, `&`, backtick, newline, `$()`, `<()`, `>()`.
- **Status:** FIXED ✅

### AUDIT-2 — stdio full-mode has no ORDER_EXEC gate
- **Files:** `tool_policy.py`, `server.py`
- **Issue:** `ALGOCHAINS_TOOL_MODE=full` + `ALGOCHAINS_REQUIRE_CONFIRMATION=0` allowed ORDER_EXEC tools without `owner_token`. Full mode is dev-only but had no enforcement.
- **Fix:** `evaluate_stdio_direct_tool` now enforces ORDER_EXEC gate with `owner_token` parity. Startup WARNING emitted once per process in full mode.
- **Status:** FIXED ✅

### AUDIT-3 — `consecutive_losses` single source of failure
- **File:** `server.py`
- **Issue:** Fills API alone: if today is a no-trade day (empty fills), returns 0.
- **Fix:** `signal_health.json` reconciliation fallback added. Both-fail: WARNING + assume 0.
- **Status:** FIXED ✅

### AUDIT-4 — Three implementation bugs in batch fixes
- `_fills_err` NameError: Python 3 deletes except-vars after block; captured as `_fills_api_err_str`.
- `__import__("os")` code smell in `tool_policy.py`: replaced with proper `import os`.
- `submit_institutional_order` broker extraction: arbitrary string used as broker name; now validated against `_KNOWN_BROKERS` set.
- **Status:** FIXED ✅
