# Remote Connectors

Use this when connecting AlgoChains MCP from Claude.ai web/mobile, API agents, or any
client that cannot spawn the local `algochains-mcp` process.

## Short Answer

For Claude.ai web/mobile custom connectors, use a public HTTPS MCP endpoint:

```text
https://<your-domain>/mcp
```

Do not use:

```text
http://localhost:8080/mcp
http://127.0.0.1:8080/mcp
http://0.0.0.0:8080/mcp
http://100.x.y.z:8080/mcp
```

Those addresses only work from the machine or private network where the server is
running. Claude.ai calls your connector from Anthropic's remote infrastructure, so it
cannot reach a local PyPI process, a phone, a LAN-only URL, or a Tailscale-only node.

## Transport Matrix

| Client | Correct transport | Connector value |
|--------|-------------------|-----------------|
| Cursor | `stdio` | Generated by `algochains-mcp --generate-config cursor` |
| Claude Desktop | `stdio` | Generated by `algochains-mcp --generate-config claude-desktop` |
| Windsurf | `stdio` | Generated by `algochains-mcp --generate-config windsurf` |
| Claude Code local | `stdio` or `http` | Local config or `claude mcp add --transport http ...` |
| Claude.ai web/mobile custom connector | Remote HTTP MCP | `https://<public-host>/mcp` |
| API agents / SDK | Custom HTTP bridge | `https://api.algochains.ai/api/mcp` plus `X-Api-Key` (not MCP Streamable HTTP) |

## Troubleshooting Cursor / IDE connection

### `No module named algochains_mcp`

Cursor spawned a Python interpreter that does not have the package installed (common on
macOS when config uses `python3 -m algochains_mcp` with Xcode Command Line Tools Python).

Fix:

```bash
pip install algochains-mcp-server
# or: pipx install algochains-mcp-server
algochains-mcp --generate-config cursor
```

Restart Cursor. The config must use `"command": "algochains-mcp"` — not `python -m
algochains_mcp`.

### streamableHttp / SSE 404 (`{"detail":"Not Found"}`)

Your `~/.cursor/mcp.json` likely has a `url` or `transport: http` entry pointing at the
wrong endpoint. Cursor speaks MCP Streamable HTTP at `POST /mcp`; the hosted developer
bridge at `https://api.algochains.ai/api/mcp` is a **different API** (SDK/CLI only).

Fix: remove `url`, `transport`, and `headers` from the `algochains` entry. Use stdio:

```json
{
  "mcpServers": {
    "algochains": {
      "command": "algochains-mcp",
      "env": { "ALGOCHAINS_TOOL_MODE": "smart" }
    }
  }
}
```

### Connected but missing subscriber / developer tools

Add credentials to the `env` block in `~/.cursor/mcp.json`:

- Subscriber paper: `ALGOCHAINS_SUBSCRIBER_KEY=sub_live_...`
- Developer tier: `ALGOCHAINS_BRIDGE_KEY=ac_live_...`

Never commit `~/.cursor/mcp.json` with real keys.

## Local Tunnel Test

Install the HTTP transport dependencies and run the MCP HTTP server locally:

```bash
pipx install "algochains-mcp-server[http]"
export ALGOCHAINS_HTTP_TRANSPORT_SECRET="<random-token>"
algochains-mcp-http --host 127.0.0.1 --port 8080
```

Expose that local server through an HTTPS tunnel:

```bash
cloudflared tunnel --url http://127.0.0.1:8080
```

Then add this URL in Claude.ai custom connector settings:

```text
https://<cloudflared-subdomain>.trycloudflare.com/mcp
```

If Claude asks for an auth token, use the exact value of
`ALGOCHAINS_HTTP_TRANSPORT_SECRET` as the bearer token.

## Production Hosted Endpoint

For real users, do not rely on a temporary tunnel. Run the HTTP transport behind a
stable HTTPS domain:

```bash
export ALGOCHAINS_HTTP_TRANSPORT_SECRET="<strong-secret>"
algochains-mcp-http --host 127.0.0.1 --port 8080
```

Put a reverse proxy in front of it, for example:

```text
https://mcp.algochains.ai/mcp -> http://127.0.0.1:8080/mcp
```

Use Cloudflare, nginx, or a managed hosting layer to enforce TLS, request size limits,
rate limits, and logging.

## Security Checklist

- Set `ALGOCHAINS_HTTP_TRANSPORT_SECRET` before exposing `/mcp`.
- Only expose read-only/demo tools for public testing.
- Never put `OWNER_API_TOKEN`, broker credentials, or `.env` values into connector docs,
  screenshots, or shared chat.
- Keep owner/live trading tools behind owner auth, WAF/IP allowlists, and explicit
  confirmation checks.
- Do not expose a Tailscale `100.x` URL as a public connector. Tailscale is for private
  node-to-node operations, not Claude.ai web/mobile.
- Prefer a stable production domain for subscribers; use tunnels only for temporary
  development tests.

## What to Tell Users

If someone asks, "What URL do I add as the custom connector on Claude mobile?", answer:

> Use the hosted HTTPS MCP endpoint, not the local PyPI process. For local testing, run
> `algochains-mcp-http --host 127.0.0.1 --port 8080`, tunnel it with Cloudflare, and use
> the tunnel's `https://.../mcp` URL. For production, use the stable hosted endpoint such
> as `https://mcp.algochains.ai/mcp`.
