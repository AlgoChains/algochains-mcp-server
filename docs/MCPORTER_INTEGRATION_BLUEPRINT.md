# MCPorter Integration Blueprint — AlgoChains Stack

> **Source:** [github.com/steipete/mcporter](https://github.com/steipete/mcporter) (3.5k stars, v0.8.1, MIT-ish)
> **What it is:** TypeScript runtime, CLI, and code-generation toolkit for MCP servers
> **Priority:** P2 — High-value DX play, not blocking launch
> **Est. effort:** 4-6 hours total across 3 phases
> **Date:** March 30, 2026

---

## TL;DR — Why This Matters

MCPorter auto-generates a **CLI + TypeScript SDK** from any MCP server's tool definitions — for free, zero code. For AlgoChains, this means:

1. **Instant CLI distribution** — `npx mcporter call algochains.place_order symbol:AAPL side:buy qty:10`
2. **Auto-generated TypeScript SDK** — typed client wrappers from our 232 tool schemas
3. **CI/CD testing** — call `algochains.server_diagnostics` in GitHub Actions without an IDE
4. **Multi-server orchestration** — compose AlgoChains + Massive + Slack in typed TS scripts
5. **OpenClaw skill enhancement** — replace bash-based MCP plumbing with typed, composable code

We already solved the "too many tools" problem with Smart Tool Discovery (V17). MCPorter solves the **"how do non-IDE consumers use our tools"** problem.

## 2. Five Integration Plays

### Play 1: Auto-Generated CLI (Free — Zero Code)

**What:** Users install AlgoChains MCP server, then immediately get CLI access via MCPorter.

**How:**
```bash
# User already has algochains-mcp configured in their IDE
# MCPorter auto-discovers it:
npx mcporter list algochains
# Output: TypeScript-style function signatures for all 232 tools

# Call any tool from terminal:
npx mcporter call algochains.place_order broker:alpaca symbol:AAPL side:buy qty:10
npx mcporter call algochains.get_positions broker:alpaca
npx mcporter call algochains.browse_marketplace asset_class:futures min_sharpe:2.0
npx mcporter call algochains.discover_tools query:"sentiment analysis"
npx mcporter call algochains.massive_search_endpoints query:"stock aggregates"
```

**Value for AlgoChains:**
- Zero development cost — MCPorter reads our MCP tool schemas and generates everything
- Terminal-first users (quants, devops) can use AlgoChains without an IDE
- Demo-friendly: `npx mcporter call algochains.browse_marketplace` in investor presentations
- Docs can show CLI examples alongside IDE examples

**Implementation:** Just document it. Add a `### Terminal / CLI` section to README.md.

### Play 2: Generated TypeScript SDK (`mcporter emit-ts`)

**What:** Auto-generate typed TypeScript client wrappers from our 232 MCP tool schemas.

**How:**
```bash
# Generate types-only .d.ts
npx mcporter emit-ts algochains --out types/algochains-tools.d.ts

# Generate full client wrapper
npx mcporter emit-ts algochains --mode client --out clients/algochains.ts
```

**Generated output (example):**
```typescript
// Auto-generated by mcporter emit-ts
export interface AlgoChainsTools {
  /**
   * Execute a trade on any connected broker.
   * @param broker Target broker (alpaca, ibkr, oanda, tradovate, ...)
   * @param symbol Ticker symbol
   * @param side buy or sell
   * @param qty Number of shares/contracts
   * @param order_type market, limit, stop, stop_limit, trailing_stop
   */
  placeOrder(broker: string, symbol: string, side: 'buy' | 'sell', qty: number, order_type?: string, limit_price?: number): Promise<PlaceOrderResult>;

  /**
   * Search validated bots by asset class, Sharpe, tier, or strategy.
   */
  browseMarketplace(asset_class?: string, min_sharpe?: number, tier?: string): Promise<MarketplaceResult>;

  /**
   * BM25 search over 232 tools. Returns matching tool names + descriptions.
   */
  discoverTools(query: string): Promise<DiscoverResult>;

  // ... 229 more tools, all typed
}
```

**Value for AlgoChains:**
- **V18 Intent Engine / Agent Swarm** can import typed AlgoChains tools
- External developers building on AlgoChains get IDE autocomplete for free
- Could publish as `@algochains/sdk` on npm (auto-generated, always in sync with MCP schema)
- No manual SDK maintenance — regenerate on every release

**Implementation:**
1. Add `mcporter emit-ts` to CI (regenerate on version bump)
2. Publish generated types as npm package
3. Reference in docs: "For TypeScript projects, install `@algochains/sdk`"

### Play 3: Standalone CLI Binary (`mcporter generate-cli`)

**What:** Compile a self-contained `algochains` CLI binary that bundles our MCP tool schemas.

**How:**
```bash
# Generate + bundle
npx mcporter generate-cli algochains --bundle dist/algochains-cli.js --name algochains

# Or compile to binary (requires Bun)
npx mcporter generate-cli algochains --compile --name algochains
```

**Result:** A standalone `algochains` CLI that users can download:
```bash
# No npm/pip required — just download and run
./algochains place_order broker:alpaca symbol:NVDA side:buy qty:5
./algochains get_portfolio_summary
./algochains browse_marketplace min_sharpe:2.0
```

**Value for AlgoChains:**
- Distribution via GitHub Releases (macOS arm64/x64, Linux)
- Marketplace creators can script submissions: `algochains submit_strategy --file my_bot.json`
- Subscribers can deploy bots from terminal: `algochains subscribe_to_bot bot_id:xxx`
- Pairs with the Python `algochains-mcp` entry point we already ship

**Implementation:**
1. Add `generate-cli` step to GitHub Actions release workflow
2. Attach compiled binaries to GitHub Releases
3. Add Homebrew formula: `brew install algochains`

### Play 4: Multi-Server Composition for OpenClaw Skills

**What:** Replace bash-based MCP tool plumbing in OpenClaw skills with typed TypeScript automations.

**Current OpenClaw skill pattern (bash + Python):**
```bash
# Skill: morning-brief
python3 -c "
import subprocess
# Call massive for market data... messy
# Call algochains for positions... messy  
# Format and post to Slack... messy
"
```

**With MCPorter (typed TypeScript):**
```typescript
import { createRuntime, createServerProxy } from "mcporter";

const runtime = await createRuntime();
const ac = createServerProxy(runtime, "algochains");
const massive = createServerProxy(runtime, "massive");
const slack = createServerProxy(runtime, "slack");

// Fetch portfolio
const positions = await ac.getPositions({ broker: "tradovate" });
const account = await ac.getAccount({ broker: "tradovate" });

// Fetch market data
const spy = await massive.callApi({
  method: "GET",
  path: "/v2/aggs/ticker/SPY/prev"
});

// Compose morning brief
const brief = `
Portfolio: $${account.json().equity}
Positions: ${positions.json().count}
SPY prev close: $${spy.json().results[0].c}
`;

// Post to Slack
await slack.postMessage({
  channel_id: "C09F415GZ6W",  // #quant-lab
  text: brief
});

await runtime.close();
```

**Value for AlgoChains:**
- Type-safe multi-server orchestration
- MCPorter auto-discovers all configured MCP servers (Massive, Slack, Notion, Pinecone, etc.)
- Connection pooling and OAuth handled automatically
- Better error handling than bash piping
- Could replace 10-20 OpenClaw skills that currently shell out to Python

**Implementation:**
1. Install MCPorter in control-tower: `npm install -g mcporter`
2. Create `config/mcporter.json` that imports Windsurf config
3. Convert highest-value OpenClaw skills to TS scripts
4. OpenClaw scheduler calls: `npx tsx skills/morning-brief.ts`

### Play 5: CI/CD Integration Testing

**What:** Test AlgoChains MCP server tools in GitHub Actions without an IDE.

**GitHub Actions workflow:**
```yaml
# .github/workflows/mcp-integration-test.yml
name: MCP Integration Tests
on: [push, pull_request]

jobs:
  test-mcp-tools:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 22 }
      - uses: actions/setup-python@v5
        with: { python-version: '3.12' }

      - name: Install AlgoChains MCP + MCPorter
        run: |
          pip install -e ".[dev]"
          npm install -g mcporter

      - name: Configure MCPorter
        run: |
          mkdir -p config
          cat > config/mcporter.json << 'EOF'
          {
            "mcpServers": {
              "algochains": {
                "command": "algochains-mcp",
                "env": { "ALGOCHAINS_TOOL_MODE": "full" }
              }
            }
          }
          EOF

      - name: Test Smart Discovery
        run: |
          mcporter call algochains.discover_tools query:"portfolio"
          mcporter call algochains.discover_tools query:"sentiment"
          mcporter call algochains.discover_tools query:"backtest"

      - name: Test Server Diagnostics
        run: mcporter call algochains.server_diagnostics --output json

      - name: Test Tool Schema Completeness
        run: |
          TOOLS=$(mcporter list algochains --json | jq '.tools | length')
          echo "Tool count: $TOOLS"
          [ "$TOOLS" -ge 200 ] || (echo "FAIL: Expected 200+ tools, got $TOOLS" && exit 1)
```

**Value for AlgoChains:**
- Regression testing for tool schemas on every PR
- Verify tool count doesn't drop (current: 232)
- Test Smart Discovery BM25 index returns relevant results
- No IDE needed — pure CLI testing

## 6. What NOT to Do

- **Don't rewrite the Python MCP server in TypeScript** — MCPorter consumes our server as-is
- **Don't add MCPorter as a runtime dependency** — it's a dev/distribution tool
- **Don't replace Windsurf MCP integration** — MCPorter supplements, doesn't replace IDE consumption
- **Don't rush Phase 3 before launch** — CLI + SDK generation are the high-value wins
